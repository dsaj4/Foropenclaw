## 二、步骤详解

### 步骤1：目录树生成

#### 1.1 操作定义
将阶段一拆解出的标题结构转化为新报告的章节框架。

#### 1.2 继承与筛选规则

##### 1.2.1 固定分析框架（强制继承）

**处理方式**：章节标题**原样保留**，不得改名或删减

**适用章节**：
- 公司概况
- 行业分析
- 业务分析
- 财务分析
- 盈利预测与估值
- 风险提示
- 投资要点

**写作要求**：
- 仅需替换主体名称（将"XX公司"改为目标公司名）
- 保持原有分析维度不变

**示例**：
```
原报告："第一章：公司概况"
新报告："第一章：公司概况"（标题不变，内容替换为目标公司）
```

##### 1.2.2 特殊分析框架（条件适配）

**筛选标准**：
对照阶段一提取的"特殊框架清单"，逐条判断目标公司是否适用：

| 判断结果 | 处理规则 | 示例 |
|---------|---------|------|
| **保留** | 目标公司符合特殊框架适用条件 | 原报告有特殊框架"多模态技术路线"，目标公司为AI企业 → **保留** |
| **删除** | 目标公司不符合适用条件 | 原报告有特殊框架"出海战略"，但目标公司海外收入占比为0 → **删除** |
| **替换** | 目标公司有对应特征但类型不同 | 原报告有特殊框架"ToB业务转型"，但目标公司始终为ToC → **替换**为"ToC用户运营" |

**命名规则**：
保持原特殊框架的**句法结构**：
- 如原框架为偏正短语"技术路线选择"
- 新框架应为"商业模式演进"
- 而非口语化表达"我们是如何赚钱的"

##### 1.2.3 具体情况总结（基于新数据重构）

**生成逻辑**：
回顾阶段一总结的"固定框架→特殊框架→情况总结"推导链，将新公司的具体数据代入该逻辑链。

**示例**：
```
原报告逻辑链：
研发投入高（固定框架）→ 技术突破快（特殊框架）→ 从追赶者变为领跑者（情况总结）

迁移应用：
若新公司研发投入占比低但商业化快：
销售投入高（固定）→ 渠道扩张快（特殊）→ 从产品力驱动转向渠道驱动（总结）
```

#### 1.3 产出物

### 步骤2：总起句撰写

#### 2.1 操作定义
基于阶段一提取的**功能类型库**和**句式模板**，根据新公司内容的逻辑需求自由组合，撰写段落开篇句。

#### 2.2 迁移方法

##### 2.2.1 功能类型库建立

从阶段一分析中，提取旧报告使用的全部总起句功能类型，形成**可选工具箱**：

| 功能类型 | 定义 | 适用场景（根据内容需求自选） |
|---------|------|------------------------|
| **宣告型** | 直接陈述本节核心结论 | 适用于观点明确、结论先行的章节（如"投资要点"、"核心结论"） |
| **概括型** | 预告本节分析维度 | 适用于多维度拆解的章节（如"财务分析"、"业务拆解"） |
| **过渡型** | 承接上文引出下文 | 适用于上下文逻辑紧密衔接的章节（如从"行业"过渡到"公司"） |

##### 2.2.2 句式模板提取

从阶段一分析中，提取各类功能类型的**句式骨架**（保留语法结构和连接词，替换实体内容）：

| 功能类型 | 原报告示例 | 提取的句式模板 | 新报告自由应用示例 |
|---------|-----------|---------------|------------------|
| **宣告型** | "X公司的核心竞争力体现在三个维度..." | "[主体]的[核心属性]体现在[数量]个维度..." | 若用于技术章节："月之暗面的技术壁垒体现在三个层面..."<br>若用于商业章节："月之暗面的变现能力体现在三个环节..." |
| **概括型** | "本章从三个维度分析..." | "本章从[数量]个[维度/方面/视角]展开..." | 若用于财务章节："本章从三个维度审视财务健康度..."<br>若用于风险章节："本节从三个层面识别潜在风险..." |
| **过渡型** | "基于前述技术积累..." | "基于前述[已论述维度]..." | 若承接行业分析："基于前述行业格局分析..."<br>若承接历史回顾："基于前述发展历程梳理..." |

##### 2.2.3 自由搭配与互文逻辑

**原则：功能自选，逻辑强制**

1. **功能与内容匹配**：
   - 所选功能类型必须适应当前段落的内容性质
   - 在需要预告结构的章节使用概括型
   - 在需要承接上下文的章节使用过渡型

2. **互文链条完整**：
   - 无论选择何种功能类型，必须确保与前后段落的逻辑衔接顺畅
   - 若前段结尾提出疑问，本段首句必须回应或承接该疑问
   - 若本段首句选择**宣告型**（直接给结论），下段首句必须处理与该结论的逻辑关系（转折、递进、或并列）

**示例：自由搭配场景**

| 场景 | 原报告选择 | 新报告可选方案 | 效果说明 |
|------|-----------|---------------|---------|
| 财务分析章首 | 概括型："本章从三个维度分析..." | **方案A（沿用概括型）**："本章从盈利能力、运营效率、资本结构三个维度审视财务表现..." | 保持原风格 |
| | | **方案B（改用宣告型）**："财务数据显示，公司正处于从投入期向盈利期过渡的关键节点..." | 直接给出财务特征判断 |
| | | **方案C（改用过渡型）**："基于前述业务高速增长的趋势，本章转向检验其财务可持续性..." | 承接业务章节，引出财务分析 |

*以上三种选择均有效，仅需确保与前后文的逻辑连贯性。*

#### 2.3 写作检查清单

撰写每个总起句后，检查：

- [ ] 功能类型与段落内容性质匹配
- [ ] 与前一段落逻辑衔接顺畅
- [ ] 句式符合模板结构
- [ ] 主体名称已替换为目标公司

### 步骤3：论证段撰写

#### 3.1 操作定义
撰写支撑总起句的正文内容，复现原报告的论证密度与风格。

#### 3.2 长论证段写作规范（Minimax风格）⭐新增

Minimax报告的核心写作特征是**"总起句+长论证段"**结构，论证深入、不使用分点列表。具体规范如下：

##### 3.2.1 段落结构：总起句 + 长论证段

**标准结构**:
```
[总起句 - 宣告型/概括型，1句话点明核心观点]
    ↓
[论证段 - 200-400字长段落，无分点，深入论证]
    ↓
[小结句 - 可选，呼应总起句]
```

**Minimax报告示例分析**:
```
【总起句 - 宣告型】
Minimax是稀缺的Pureplay大模型技术公司，坚持生而全球化（Born Global）策略。

【长论证段 - 约300字，无分点】
公司自创立之初就定位为全球化的AI公司，聚焦先进模型以及AI原生产品的开发，
根植于底层较强的模型能力，已为来自超过200个国家及地区的超2亿个人用户，
以及来自超100个国家及地区的超10万家企业以及开发者提供服务。模型端，
公司持续迭代升级大模型能力以及开发新模型，目前已形成了以M2、Hailuo-02、
Speech-02等为核心的全模态大模型组合。公司在模型层的演进方向将更多体现
在多模态能力的进一步整合，以及在统一模型框架下持续提升效率与稳定性。
随着全模态统一模型逐步落地，模型能力有望从单一模态的局部优势转向系统
层面的整体提升；而既有的高效基础架构，也将为后续的大规模商业化应用
提供空间。

【小结句 - 可选】
产品端，公司主营产品包含多元化C端原生产品以及B端开放平台...
```

##### 3.2.2 避免分点列表的替代写法

**❌ 避免使用**:
- 项目符号列表（•、-、*）
- 编号列表（1. 2. 3.）
- 表格（除非数据密集型内容）
- 短句堆砌

**✅ 改用长句复合结构**:

| 分点写法 | 长论证段替代写法 |
|---------|----------------|
| 公司有三个优势：• 技术领先 • 产品丰富 • 团队优秀 | 公司在技术、产品与团队三个维度形成了综合竞争优势。技术层面，...；产品层面，...；团队层面，... |
| 营收增长原因：1. 用户增加 2. 单价提升 3. 新品推出 | 营收增长源于用户规模扩张、客单价提升与新产品贡献三重因素的共振。用户方面，...；价格方面，...；产品方面，... |

**转换技巧**:
1. **使用复合句**: 将多个短句用分号、逗号连接成长句
2. **使用连接词**: "一方面...另一方面..."、"首先...进而...最终..."
3. **使用同位语**: "公司形成了三大核心能力——技术、产品与市场——共同支撑增长"
4. **使用插入语**: "公司在模型层，具体而言是在多模态能力整合方面，持续投入"

##### 3.2.3 论证深入的四个层次

长论证段必须包含以下四个层次的深度融合：

```
层次1：事实陈述（数据/现象）
    ↓
层次2：原因分析（为什么会出现）
    ↓
层次3：影响评估（意味着什么）
    ↓
层次4：趋势判断（未来会怎样）
```

**示例**（Minimax风格）:
```
公司营收从2023年的346万美金大幅提升至2024年的3052万美金，CAGR达782%
【事实陈述】，这一高速增长源于公司在商业化之初便找到了"可规模化变现"的
ToC产品形态，Talkie/星野凭借情感对话、角色扮演等功能构建了明显优于
同类产品的用户体验，从而能够较早跑通订阅和增值付费模型【原因分析】，
实现用户规模与收入增长的正反馈，为商业化奠定了坚实基础【影响评估】。
2025年前三季度，营收进一步增长至5344万美金，同比+175%，反映出商业化
能力持续增强【趋势判断】。
```

##### 3.2.4 论据与观点的融合技巧

**不要**：先列数据，再下结论（割裂式）
```
❌ 公司营收100亿，增长20%。这表明公司发展良好。
```

**要**：数据嵌入观点，融为一体（融合式）
```
✅ 公司营收突破100亿元、同比增长20%的双高表现，反映出产品市场匹配度
持续提升，规模效应开始显现。
```

**融合句式模板**:

| 模板 | 示例 |
|------|------|
| "[数据]的[指标特征]，[观点]" | "营收增速从20%提升至35%的加速态势，反映出竞争壁垒正在形成" |
| "[数据]表明/显示/反映出[观点]" | "毛利率从12%提升至23%的持续改善，表明单位经济模型正在优化" |
| "[观点]，体现为[数据]" | "公司全球化战略成效显著，体现为海外收入占比从19%提升至73%" |
| "与[对比对象]相比，[数据]凸显出[观点]" | "与行业平均增速15%相比，公司35%的增长凸显出差异化优势" |

##### 3.2.5 长论证段的节奏控制

**段落长度标准**:
- 常规论证段：200-300字
- 重点论证段：300-400字
- 超长论证段：不超过500字（超过需拆分）

**节奏变化技巧**:
1. **数据密度变化**: 连续2-3个数据后，插入分析判断句
2. **句式长短变化**: 长句（30-40字）后接中等句（20-25字），避免单调
3. **论证深度变化**: 从事实→原因→影响→趋势，逐层递进

**检查清单**:
- [ ] 段落长度200-400字
- [ ] 无项目符号或编号列表
- [ ] 包含事实+原因+影响+趋势四个层次
- [ ] 数据与观点融合呈现
- [ ] 使用复合句和连接词保持流畅

#### 3.3 信息密度配比

根据阶段一分析的"事实论据 vs 分析判断"比例，控制新报告每段内容构成：

| 原报告论证段类型 | 事实:观点比例 | 新报告写作指令 |
|----------------|-------------|--------------|
| 数据密集型（如财务分析） | 7:3 | 每段7句陈述数据（营收、增速、占比），3句分析原因/影响 |
| 观点密集型（如行业趋势） | 3:7 | 每段3句列举行业现象，7句展开逻辑推演与趋势判断 |
| 平衡型（如竞争格局） | 5:5 | 交替使用"事实陈述→分析判断"模式（一句数据一句解读） |

#### 3.3 论证逻辑复刻

沿用阶段一识别的论证模式：

| 论证模式 | 复刻要求 | 示例 |
|---------|---------|------|
| **归纳法** | 若原报告通过"列举3个产品案例→总结技术领先"论证，新报告必须同样列举3个业务证据，再归纳结论 | 原：产品A+产品B+产品C→技术领先<br>新：业务X+业务Y+业务Z→竞争力强 |
| **演绎法** | 若原报告使用"大前提→小前提→结论"结构，新报告必须保持相同逻辑链条 | 原：行业高增长+公司市占率提升→业绩将超预期<br>新：行业整合+公司成本优势→盈利能力将改善 |
| **比较法** | 若原报告使用"横向对比（vs竞品）+纵向对比（vs历史）"双维度论证，新报告必须同样构建双维度比较框架 | 原：vs竞品（技术领先）+vs历史（增速提升）<br>新：vs竞品（成本更低）+vs历史（份额提升） |

#### 3.4 信源标注规范

严格遵循阶段一标注的"事实陈述必须附加信源"规则：

| 数据类型 | 标注规则 | 示例 |
|---------|---------|------|
| 财务数据 | 必须标注来源 | "营收100亿元[来源：公司2023年报]" |
| 行业数据 | 保持相同模糊度 | 若原报告用"第三方统计"，新报告同类数据保持相同模糊度 |
| 预测数据 | 需说明假设 | "基于XX假设，我们预计..." |

**重要原则**：
- 若阶段一分析发现原报告对财务数据均标注"[来源：公司年报]"，新报告所有财务数据必须同样标注来源（不得出现无来源的数据）
- 若阶段一发现原报告对行业数据使用"第三方统计"模糊处理，新报告同类数据保持相同模糊度（不得过度精确化无法验证的数据）

### 步骤4：语言风格校准

#### 4.1 句长强制对齐

测量阶段一得出的"平均句长"，新报告写作时控制偏差在±10%以内：

| 原报告平均句长 | 新报告控制范围 | 调整方法 |
|---------------|---------------|---------|
| 22字 | 20-24字 | 长句拆分（超过24字必须拆分为两句）；短句合并（低于20字的短句需与前句合并或扩充定语） |

**调整示例**：
```
过长句（28字）：公司2023年实现营业收入100亿元同比增长20%净利润15亿元
拆分后：公司2023年实现营业收入100亿元，同比增长20%；净利润达15亿元。

过短句（12字）：公司营收增长。主要得益于产品销量提升。
合并后：公司营收增长主要得益于产品销量的显著提升。
```

#### 4.2 术语密度控制

统计阶段一的"行业术语占比"，新报告保持相同密度：

| 原报告术语密度 | 新报告目标 | 控制方法 |
|---------------|-----------|---------|
| 15%（每100字15个专业术语） | 13-17% | 避免口语化稀释或过度学术化 |

**术语使用原则**：
- 优先使用行业通用术语
- 首次出现需简要解释
- 保持全文术语使用一致性

#### 4.3 客观度评级对齐

根据阶段一的"客观度评分"调整主观表达强度：

| 原报告客观度 | 等级 | 新报告写作要求 |
|-------------|------|---------------|
| 0.85（高客观） | 高 | 禁用"我们认为"、"显然"、"无疑"等主观表述，改用"数据显示"、"值得注意的是"、"从侧面印证"等客观引导词 |
| 0.70（中客观） | 中 | 可适当使用"这反映了"、"体现了"等轻度判断词，但避免"必然"、"肯定"等强判断 |
| 0.55（低客观） | 低 | 可使用主观判断，但需有数据支撑 |

**主观词汇分级表**：

| 禁用（高客观） | 慎用（中客观） | 推荐（全等级） |
|---------------|---------------|---------------|
| 必然 | 反映了 | 数据显示 |
| 肯定 | 体现了 | 值得注意的是 |
| 毫无疑问 | 说明 | 从侧面印证 |
| 显然 | 意味着 | 根据统计 |
| 绝对 | 表明 | 分析指出 |
