> **流程定位**：验证新报告与阶段一提取标准的符合度

## 二、校验维度详解

### 维度1：结构保真校验

#### 1.1 固定框架完整性检查

**检查项**：新报告是否遗漏阶段一标注的固定框架

**合格标准**：
- 固定框架章节全部存在
- 无遗漏、无删减

**检查方法**：
```markdown
| 标准固定框架 | 新报告对应章节 | 状态 |
|-------------|---------------|------|
| 公司概况 | 第一章：公司概况 | ✓ 存在 |
| 行业分析 | 第二章：行业分析 | ✓ 存在 |
| 业务分析 | 第三章：业务分析 | ✓ 存在 |
| 财务分析 | 第四章：财务分析 | ✓ 存在 |
| 风险提示 | 第七章：风险提示 | ✓ 存在 |
| 投资要点 | 第八章：投资要点 | ✓ 存在 |
```

**评分规则**：
- 全部存在：100分
- 每遗漏1个：-20分

#### 1.2 特殊框架适配性检查

**检查项**：特殊框架的选择是否合理的

**合格标准**：
- 保留的特殊框架符合适用条件
- 删除的特殊框架确实不适用
- 替换的特殊框架命名规范

**检查方法**：
```markdown
| 原报告特殊框架 | 新报告处理 | 适配理由 | 评估 |
|---------------|-----------|---------|------|
| 技术路线分析 | 保留 | 目标公司研发投入15%，符合适用条件 | ✓ 合理 |
| 出海战略 | 删除 | 目标公司海外收入占比0%，不适用 | ✓ 合理 |
| ToB转型 | 替换为ToC运营 | 目标公司始终为ToC模式 | ✓ 合理 |
```

**评分规则**：
- 全部合理：100分
- 每处不合理：-25分

#### 1.3 章节标题变异度检查

**检查项**：章节标题与标准框架的差异程度

**合格标准**：
- 标题变异度 < 20%（字数/词性变化）

**计算方法**：
```
变异度 = (标题字数变化 + 关键词替换数) / 原标题长度 × 100%

示例：
原标题："财务分析"（4字）
新标题："财务状况深度解析"（8字）
字数变化：4字
关键词替换："分析"→"解析"（1处）
变异度 = (4 + 1) / 4 × 100% = 125% ❌（超标）
```

**评分规则**：
- 变异度 < 10%：100分
- 变异度 10-20%：80分
- 变异度 20-30%：60分
- 变异度 > 30%：40分

---

### 维度2：逻辑保真校验

#### 2.1 总起句逻辑连贯性检查

**检查项**：新报告总起句间的逻辑衔接是否顺畅

**合格标准**：
- 段落间无逻辑断层
- 前后段落逻辑关系清晰

**检查方法**：
逐段检查首句与前一段落的逻辑关系：

```markdown
| 段落位置 | 前段结尾 | 本段首句 | 逻辑关系 | 评估 |
|---------|---------|---------|---------|------|
| 第2章首 | - | 概括型 | 开启新章 | ✓ |
| 第2章末 | "行业增速放缓" | 过渡型 | 引出应对策略 | ✓ |
| 第3章首 | - | 宣告型 | 直接给出业务特征 | ✓ |
| 第3章第2段 | "业务多元化" | 概括型 | 预告拆解维度 | ✓ |
```

**评分规则**：
- 全部连贯：100分
- 每处断层：-15分

#### 2.2 互文逻辑完整性检查

**检查项**：总起句是否回应前文提出的问题或结论

**合格标准**：
- 前段提出疑问，本段首句回应或承接
- 本段宣告结论，下段处理逻辑关系

**检查方法**：
```markdown
| 检查点 | 要求 | 新报告表现 | 评估 |
|--------|------|-----------|------|
| 疑问回应 | 前段提出疑问，本段需回应 | 第2章末提出"如何应对？"，第3章首回应"公司通过..." | ✓ |
| 结论承接 | 宣告型结论需有后续处理 | 第4章首宣告"财务健康"，第4章第2段分析"具体指标..." | ✓ |
```

**评分规则**：
- 全部完整：100分
- 每处缺失：-20分

---

### 维度3：密度保真校验

#### 3.1 事实:观点比例检查

**检查项**：新报告论证段的事实:观点比例是否符合阶段一标准

**合格标准**：
- 各章节比例误差 < 15%

**检查方法**：
```markdown
| 章节 | 阶段一标准 | 新报告实测 | 误差 | 评估 |
|------|-----------|-----------|------|------|
| 财务分析 | 7:3 | 6.5:3.5 | -7% | ✓ 合格 |
| 行业趋势 | 3:7 | 3.5:6.5 | +8% | ✓ 合格 |
| 竞争格局 | 5:5 | 4.5:5.5 | +10% | ✓ 合格 |
```

**评分规则**：
- 误差 < 10%：100分
- 误差 10-15%：80分
- 误差 15-20%：60分
- 误差 > 20%：40分

#### 3.2 论证模式一致性检查

**检查项**：新报告是否沿用阶段一识别的论证模式

**合格标准**：
- 同类型章节使用相同论证模式

**检查方法**：
```markdown
| 章节 | 阶段一模式 | 新报告模式 | 一致性 | 评估 |
|------|-----------|-----------|--------|------|
| 财务分析 | 归纳法 | 归纳法 | ✓ | 一致 |
| 竞争格局 | 比较法 | 比较法 | ✓ | 一致 |
| 行业趋势 | 演绎法 | 归纳法 | ✗ | 不一致 |
```

**评分规则**：
- 全部一致：100分
- 每处不一致：-20分

#### 3.3 信源标注规范性检查

**检查项**：信源标注是否符合阶段一规范

**合格标准**：
- 财务数据均有来源标注
- 行业数据标注方式一致
- 无来源数据占比 < 5%

**检查方法**：
```markdown
| 数据类型 | 阶段一规范 | 新报告表现 | 评估 |
|---------|-----------|-----------|------|
| 财务数据 | 必须标注来源 | 100%标注 | ✓ |
| 行业数据 | 可模糊处理 | 保持模糊度一致 | ✓ |
| 无来源数据 | 占比<5% | 占比3% | ✓ |
```

**评分规则**：
- 全部规范：100分
- 无来源数据占比5-10%：80分
- 无来源数据占比10-15%：60分
- 无来源数据占比 > 15%：40分

---

### 维度4：风格保真校验

#### 4.1 句长偏差检查

**检查项**：新报告句长是否在阶段一基准范围内

**合格标准**：
- 平均句长偏差 < 10%

**检查方法**：
```markdown
| 指标 | 阶段一标准 | 新报告实测 | 偏差 | 评估 |
|------|-----------|-----------|------|------|
| 平均句长 | 22字 | 21字 | -4.5% | ✓ 合格 |
| 最长句 | 45字 | 42字 | -6.7% | ✓ 合格 |
| 最短句 | 8字 | 9字 | +12.5% | ✓ 合格 |
```

**评分规则**：
- 偏差 < 5%：100分
- 偏差 5-10%：90分
- 偏差 10-15%：75分
- 偏差 > 15%：60分

#### 4.2 术语密度检查

**检查项**：新报告术语密度是否与阶段一标准一致

**合格标准**：
- 术语密度偏差 < 10%

**检查方法**：
```markdown
| 章节 | 阶段一标准 | 新报告实测 | 偏差 | 评估 |
|------|-----------|-----------|------|------|
| 整体 | 15% | 16% | +6.7% | ✓ 合格 |
| 财务章节 | 18% | 19% | +5.6% | ✓ 合格 |
| 业务章节 | 10% | 11% | +10% | ✓ 合格 |
```

**评分规则**：
- 偏差 < 5%：100分
- 偏差 5-10%：90分
- 偏差 10-15%：75分
- 偏差 > 15%：60分

#### 4.3 客观度评分检查

**检查项**：新报告客观度评分是否达到阶段一标准

**合格标准**：
- 客观度评分 ≥ 阶段一标准
- 禁用词汇出现次数 = 0

**检查方法**：
```markdown
| 指标 | 阶段一标准 | 新报告实测 | 评估 |
|------|-----------|-----------|------|
| 客观度评分 | 0.85 | 0.87 | ✓ 达标 |
| 禁用词出现 | 0次 | 0次 | ✓ 达标 |
| 主观词密度 | <10% | 8% | ✓ 达标 |
```

**评分规则**：
- 达标且优于标准：100分
- 达标：90分
- 低于标准 < 0.05：75分
- 低于标准 ≥ 0.05：60分

---

### 维度5：功能适配校验

#### 5.1 总起句功能类型匹配度检查

**检查项**：总起句功能类型是否与段落内容性质匹配

**合格标准**：
- 宣告型用于结论先行章节
- 概括型用于多维拆解章节
- 过渡型用于承上启下章节

**检查方法**：
```markdown
| 段落 | 功能类型 | 内容性质 | 匹配度 | 评估 |
|------|---------|---------|--------|------|
| 投资要点 | 宣告型 | 结论先行 | ✓ 匹配 | 正确 |
| 财务分析 | 概括型 | 多维拆解 | ✓ 匹配 | 正确 |
| 业务章节衔接 | 过渡型 | 承上启下 | ✓ 匹配 | 正确 |
| 风险提示 | 宣告型 | 清单式 | ✗ 不匹配 | 应为概括型 |
```

**评分规则**：
- 全部匹配：100分
- 每处不匹配：-15分
