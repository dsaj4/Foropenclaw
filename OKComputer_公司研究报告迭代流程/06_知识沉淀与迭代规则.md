# 阶段四：知识沉淀与迭代规则

> **流程定位**：将本次迭代经验转化为可复用的结构化资产  
> **输入**：本次迭代全部产出（框架要素清单、新报告初稿、一致性校验报告）  
> **输出**：更新的框架资产（框架库新版本、新增特殊框架、优化参数）

---

## 二、步骤详解

### 步骤1：沉淀价值评估

#### 1.1 可沉淀要素识别

从本次迭代产出中，识别以下类型的可沉淀要素：

| 要素类型 | 来源 | 示例 |
|---------|------|------|
| **新特殊框架** | 新标杆报告发现的特殊章节 | "平台生态分析"框架 |
| **优化句式模板** | 新报告验证有效的句式 | 新的宣告型模板 |
| **调整风格参数** | 多报告验证后的参数优化 | 句长从22字调整为20字 |
| **新增论证模式** | 新报告使用的有效论证方式 | 因果链论证模式 |
| **修订适用条件** | 特殊框架适用条件的细化 | "研发型企业"定义补充 |

#### 1.2 通用性评估标准

对每个可沉淀要素，评估其通用性和可复用性：

| 评估维度 | 评估标准 | 评分 |
|---------|---------|------|
| **适用范围** | 适用于多少比例的目标公司 | 1-5分 |
| **验证次数** | 已在多少份报告中验证有效 | 1-5分 |
| **稳定性** | 参数/框架是否稳定有效 | 1-5分 |
| **易用性** | 其他Agent能否理解和应用 | 1-5分 |

**综合评分 = 四项评分平均值**

| 综合评分 | 沉淀决策 | 处理方式 |
|---------|---------|---------|
| 4.0-5.0 | 高价值 | 纳入框架库主版本 |
| 3.0-3.9 | 中价值 | 纳入框架库候选，标记待验证 |
| 2.0-2.9 | 低价值 | 作为参考案例，不纳入框架库 |
| <2.0 | 无价值 | 不沉淀 |

#### 1.3 沉淀决策矩阵

```
                    高通用性
                       │
         ┌─────────────┼─────────────┐
         │   纳入主库   │   纳入主库   │
         │   立即生效   │   优先扩展   │
  高验证 │             │             │ 低验证
  次数   │   ──────────┼─────────────  次数
         │   纳入候选   │   暂不纳入   │
         │   待验证    │   持续观察   │
         └─────────────┼─────────────┘
                       │
                    低通用性
```

### 步骤2：框架库更新

#### 2.1 固定框架更新

**更新原则**：
- 固定框架原则上保持稳定
- 仅在以下情况下更新：
  1. 发现行业通用标准变化
  2. 多份报告验证原有框架缺失关键维度
  3. 监管要求变化

**更新流程**：
```
发现更新需求
      │
      ▼
评估影响范围（影响所有报告）
      │
      ▼
制定更新方案
      │
      ▼
主版本号+1（如v1.x → v2.0）
      │
      ▼
更新所有引用该框架的报告
```

#### 2.2 特殊框架库扩展

**扩展条件**：
- 新发现的特殊框架综合评分 ≥ 3.0
- 与现有特殊框架不重复
- 适用条件清晰可判定

**扩展流程**：
```markdown
## 新增特殊框架记录

**框架名称**：[名称]
**框架编号**：框架G（按字母顺序递增）
**适用条件**：[详细描述]
**章节结构**：
```
第五章：[框架名称]
├── 5.1 [子章节1]
├── 5.2 [子章节2]
└── ...
```
**来源报告**：[标杆报告名称]
**验证状态**：待验证（1/3）
**创建日期**：[日期]
**创建版本**：[框架库版本]
```

**特殊框架验证机制**：

| 验证状态 | 含义 | 升级条件 |
|---------|------|---------|
| 待验证（1/3） | 刚发现，需验证 | 在1份新报告中成功复用 |
| 待验证（2/3） | 已验证1次 | 在2份新报告中成功复用 |
| 已验证（3/3） | 已验证2次 | 在3份新报告中成功复用 |
| 正式框架 | 验证完成 | - |

#### 2.3 风格参数优化

**优化原则**：
- 参数调整需基于多份报告验证
- 单次报告发现的参数偏差不直接调整标准
- 记录参数变化趋势

**参数优化流程**：
```markdown
## 参数优化记录

**参数名称**：平均句长
**当前标准**：22字
**本次报告实测**：20字
**历史趋势**：[22, 21, 20, 21, 20]（最近5次）
**趋势判断**：呈下降趋势
**优化建议**：考虑将标准调整为21字
**优化时机**：再观察2-3次报告后决定
**记录日期**：[日期]
```

### 步骤3：版本管理

#### 3.1 版本号规则

采用语义化版本控制（Semantic Versioning）：

```
版本格式：主版本号.次版本号.修订号（如v1.2.3）

主版本号（Major）：
- 固定框架重大调整
- 章节结构变更
- 不兼容的框架变化
- 变更示例：v1.x → v2.0

次版本号（Minor）：
- 特殊框架库扩展
- 新增特殊框架
- 向下兼容的功能添加
- 变更示例：v1.1 → v1.2

修订号（Patch）：
- 风格参数微调
- 句式模板优化
- 文档错误修正
- 变更示例：v1.1.1 → v1.1.2
```

#### 3.2 版本变更判定

| 变更类型 | 版本号变化 | 示例 |
|---------|-----------|------|
| 新增特殊框架 | 次版本号+1 | v1.0 → v1.1 |
| 特殊框架转正 | 次版本号+1 | v1.1 → v1.2 |
| 参数微调 | 修订号+1 | v1.1.0 → v1.1.1 |
| 固定框架调整 | 主版本号+1 | v1.x → v2.0 |

#### 3.3 变更日志规范

```markdown
# 变更日志

## [版本号] - [日期]

### 新增
- 新增特殊框架"[框架名称]"，适用于[适用条件]
- 新增句式模板[类型]模板[X]号

### 优化
- 优化[参数名称]标准，从[旧值]调整为[新值]
- 优化[框架名称]的适用条件描述

### 修复
- 修复[问题描述]

### 废弃
- 废弃[框架/参数名称]，原因：[原因]

---

## [旧版本号] - [日期]
[历史变更记录...]
```

## 三、持续迭代机制

### 3.1 迭代触发条件

| 触发场景 | 执行流程 | 预期产出 |
|---------|---------|---------|
| 分析新标杆报告 | 阶段一 → 阶段四 | 框架库更新 |
| 使用框架写新报告 | 阶段二 → 阶段三 | 新报告 + 校验记录 |
| 框架库版本过期 | 阶段一（重新校准）→ 阶段四 | 框架库升级 |
| 积累足够迭代记录 | 阶段四（批量分析） | 参数优化 |

### 3.2 迭代频率建议

| 迭代类型 | 建议频率 | 说明 |
|---------|---------|------|
| 框架库小版本更新 | 每2-3次报告分析 | 新增特殊框架、参数微调 |
| 框架库大版本更新 | 每季度或重大变更 | 固定框架调整 |
| 参数批量优化 | 每10次报告分析 | 基于趋势分析优化参数 |
| 知识图谱更新 | 持续更新 | 每次迭代后更新关联关系 |

### 3.3 迭代质量监控

建立质量指标监控体系：

| 监控指标 | 健康阈值 | 异常处理 |
|---------|---------|---------|
| 平均一致性评分 | ≥80分 | 低于阈值检查框架库是否需要校准 |
| 框架复用率 | ≥70% | 低于阈值检查特殊框架适用条件 |
| 修订返工率 | ≤20% | 高于阈值检查阶段二执行质量 |
| 新框架采纳率 | 30-50% | 过高可能说明框架库不完善 |

## 五、快速参考：沉淀决策检查清单

完成一次迭代后，对照以下清单决定是否沉淀：

### 5.1 新特殊框架检查

- [ ] 该框架是否在多份报告中出现？
- [ ] 适用条件是否清晰可判定？
- [ ] 与现有特殊框架是否重复？
- [ ] 是否有公开数据支撑该框架分析？
- [ ] 综合评分是否 ≥ 3.0？

**决策**：
- 5项全选 → 纳入框架库，次版本号+1
- 4项选中 → 纳入候选库，标记待验证
- ≤3项选中 → 暂不纳入

### 5.2 风格参数调整检查

- [ ] 该参数偏差是否在多份报告中出现？
- [ ] 偏差是否呈现一致趋势？
- [ ] 调整是否会影响已有报告？
- [ ] 调整后是否更符合行业通用标准？

**决策**：
- 4项全选 → 调整参数，修订号+1
- 3项选中 → 记录趋势，暂不调整
- ≤2项选中 → 保持现状

### 5.3 句式模板新增检查

- [ ] 该句式是否在多份报告中验证有效？
- [ ] 句式结构是否清晰可复用？
- [ ] 是否填补了现有模板的空白？

**决策**：
- 3项全选 → 纳入模板库
- 2项选中 → 记录待观察
- ≤1项选中 → 暂不纳入
