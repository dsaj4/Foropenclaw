# 快速入门指南

> **目标**：5分钟理解核心流程，30分钟完成首次使用

---

## 第一步：理解核心概念（5分钟）

### 1.1 什么是"框架"？

**框架 = 报告的章节结构 + 写作风格 + 论证方式**

就像做菜的菜谱：
- **固定框架** = 必放的基础调料（盐、油）
- **特殊框架** = 根据菜品选择的特色调料（辣椒、咖喱）
- **风格参数** = 火候、刀工等技艺标准

### 1.2 四阶段流程一句话解释

| 阶段 | 一句话解释 | 类比 |
|------|-----------|------|
| **阶段一** | 拆解一份好报告，提取它的"菜谱" | 学做菜时先看大师怎么做 |
| **阶段二** | 用提取的"菜谱"做新菜 | 按菜谱做给客人吃 |
| **阶段三** | 检查新菜是否符合标准 | 尝味道，看卖相 |
| **阶段四** | 把新经验加入"菜谱大全" | 记录改良配方 |

## 第三步：场景A完整示例（首次建立框架库）

### 准备材料

1. 一份优质标杆报告（如"腾讯控股研究报告"）
2. 目标公司数据（如"字节跳动"的公开信息）

### 执行流程

#### Step 1：执行阶段一（框架分析）

**目标**：拆解标杆报告，提取框架要素

**核心任务**：
```markdown
1. 提取目录结构
   - 记录所有章节标题
   - 标注固定框架 vs 特殊框架

2. 分析总起句
   - 找出每章第一段第一句
   - 分类：宣告型/概括型/过渡型

3. 拆解论证逻辑
   - 财务章节：数据多还是观点多？
   - 行业章节：用什么论证方式？

4. 量化语言风格
   - 平均句长多少字？
   - 术语占比多少？
   - 主观词汇多吗？
```

**产出物**：`框架要素清单_腾讯控股_20260304.md`

---

#### Step 2：执行阶段二（框架重构写作）

**目标**：用提取的框架写新报告

**核心任务**：
```markdown
1. 生成目录树
   - 100%继承固定框架
   - 筛选特殊框架（保留/删除/替换）

2. 撰写总起句
   - 从功能库选择类型
   - 套用句式模板
   - 确保逻辑连贯

3. 撰写论证段
   - 控制信息密度（数据:观点比例）
   - 复刻论证模式
   - 标注数据来源

4. 校准语言风格
   - 句长对齐（±10%）
   - 术语密度一致
   - 客观度达标
```

**产出物**：`新报告初稿_字节跳动_20260304.md`

---

#### Step 3：执行阶段三（一致性校验）

**目标**：检查新报告是否符合标准

**核心任务**：
```markdown
检查5个维度：
1. 结构保真：固定框架都保留了吗？
2. 逻辑保真：段落间逻辑连贯吗？
3. 密度保真：信息比例符合吗？
4. 风格保真：句长/术语/客观度达标吗？
5. 功能适配：总起句类型匹配吗？

计算综合得分：
- ≥85分：通过
- 70-84分：修订后通过
- <70分：返回重写
```

**产出物**：`一致性校验报告_字节跳动_20260304.md`

---

#### Step 4：执行阶段四（知识沉淀）

**目标**：将经验转化为可复用资产

**核心任务**：
```markdown
1. 评估沉淀价值
   - 发现新特殊框架了吗？
   - 参数需要调整吗？
   - 句式模板需要新增吗？

2. 更新框架库
   - 新增特殊框架 → 次版本号+1
   - 调整参数 → 修订号+1
   - 固定框架变更 → 主版本号+1

3. 记录变更日志
   - 版本号变化
   - 变更内容
   - 变更原因
```

**产出物**：`报告结构框架_v1.0.md`（初始化版本）

## 第四步：场景B快速示例（使用框架写报告）

### 前提

已有 `报告结构框架_v1.0.md`

### 执行流程

```markdown
# 跳过阶段一，直接执行阶段二

## 阶段二：框架重构写作

### 1. 加载框架库
- 读取报告结构框架_v1.0.md
- 提取固定框架清单
- 提取特殊框架库

### 2. 生成新报告目录
- 100%继承固定框架
- 根据目标公司特征筛选特殊框架
- 标注继承来源

### 3. 逐章撰写
- 每章先写总起句（选功能类型→套模板）
- 再写论证段（控密度→复刻逻辑→标信源）
- 最后校准风格（句长→术语→客观度）

### 4. 产出初稿
- 新报告初稿_[目标公司]_[日期].md

## 阶段三：一致性校验

- 按5个维度检查
- 计算综合得分
- 通过则完成，不通过则修订
```

## 第六步：常见问题

### Q1：第一次应该选什么标杆报告？

**A**：选择行业头部公司的券商研报，结构完整、分析规范。推荐：
- 互联网：腾讯、阿里
- 消费：茅台、五粮液
- 科技：宁德时代、比亚迪

### Q2：特殊框架怎么判断是否保留？

**A**：对照适用条件判断：
- 技术路线分析 → 研发投入>10%？
- 平台生态分析 → 有双边市场？
- 出海战略 → 海外收入>20%？

### Q3：一致性评分低于70分怎么办？

**A**：返回阶段二重写，重点检查红色项（单项<70分的维度）。

### Q4：多久更新一次框架库？

**A**：
- 小更新（新增特殊框架）：每2-3次报告
- 大更新（固定框架调整）：每季度
- 参数优化：每10次报告批量分析

### Q5：可以跳过阶段一吗？

**A**：可以，如果已有框架库，直接使用阶段二写新报告。
