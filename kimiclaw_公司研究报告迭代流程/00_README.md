# 公司研究报告分析写作迭代流程

> **版本**：v1.0  
> **创建日期**：2026-03-04  
> **适用场景**：公司研究报告的自动化/半自动化撰写与持续优化

---

## 一、流程体系概览

本流程是一套**可积累、可迭代**的公司研究报告分析写作系统，通过"分析-提取-复用-沉淀"的闭环机制，实现报告写作能力的持续进化。

### 1.1 核心流程架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    报告分析写作迭代总流程                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │  阶段一：     │───→│  阶段二：     │───→│  阶段三：     │      │
│  │  框架分析     │    │  框架重构     │    │  一致性校验   │      │
│  │  (解构)       │    │  写作(复现)   │    │  (验证)       │      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
│         │                   │                   │               │
│         ↓                   ↓                   ↓               │
│  ┌──────────────────────────────────────────────────────┐      │
│  │              阶段四：知识沉淀与框架迭代                  │      │
│  │         (将新经验转化为可复用的结构化资产)               │      │
│  └──────────────────────────────────────────────────────┘      │
│                              │                                   │
│                              ↓                                   │
│                    ┌─────────────────┐                          │
│                    │  更新报告结构框架  │ ←──── 持续积累循环      │
│                    │  (版本迭代)       │                          │
│                    └─────────────────┘                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 四阶段核心任务

| 阶段 | 核心任务 | 输入 | 输出 |
|------|---------|------|------|
| **阶段一** | 解构标杆报告，提取可复用要素 | 标杆研究报告 | 框架要素清单 |
| **阶段二** | 基于提取要素撰写新报告 | 新公司数据 + 框架要素清单 | 新报告初稿 |
| **阶段三** | 验证新报告与提取标准的符合度 | 新报告初稿 + 框架要素清单 | 一致性校验报告 |
| **阶段四** | 将新经验转化为可复用资产 | 本次迭代全部产出 | 更新的框架库 |

## 三、使用场景指南

### 场景1：首次建立框架库

**适用情况**：第一次使用本流程，没有任何历史框架

**执行步骤**：
```
1. 准备一份优质标杆报告（如"腾讯控股研究报告"）
2. 执行阶段一：框架分析流程
3. 执行阶段二：框架重构写作（选择目标公司）
4. 执行阶段三：一致性校验
5. 执行阶段四：知识沉淀（初始化框架库v1.0）
```

**预期产出**：
- 框架要素清单_[标杆公司]_[日期].md
- 新报告初稿_[目标公司]_[日期].md
- 报告结构框架_v1.0.md（初始化版本）

---

### 场景2：使用已有框架撰写新报告

**适用情况**：已有框架库，需要撰写新的公司研究报告

**执行步骤**：
```
1. 准备目标公司数据
2. 加载现有框架库（报告结构框架_vX.X.md）
3. 执行阶段二：框架重构写作
4. 执行阶段三：一致性校验
5. （可选）执行阶段四：如有新发现则更新框架库
```

**预期产出**：
- 新报告初稿_[目标公司]_[日期].md
- 一致性校验报告_[目标公司]_[日期].md

---

### 场景3：分析新标杆报告并更新框架库

**适用情况**：发现新的优质标杆报告，希望提取新框架要素

**执行步骤**：
```
1. 准备新标杆报告
2. 执行阶段一：框架分析流程
3. 对比现有框架库，识别新增/优化点
4. 执行阶段四：知识沉淀（更新框架库版本）
```

**预期产出**：
- 框架要素清单_[新标杆公司]_[日期].md
- 报告结构框架_vX.X.md（更新版本）

---

### 场景4：框架库版本升级

**适用情况**：框架库使用一段时间后，需要批量优化

**执行步骤**：
```
1. 回顾最近N次迭代的记录
2. 分析参数变化趋势
3. 识别需要调整的框架/参数
4. 执行阶段四：批量知识沉淀
5. 发布新版本框架库
```

## 四、核心概念解释

### 4.1 固定框架 vs 特殊框架

| 类型 | 定义 | 继承规则 | 示例 |
|------|------|---------|------|
| **固定框架** | 所有公司报告必须包含的基础章节 | 强制100%继承 | 公司概况、财务分析、风险提示 |
| **特殊框架** | 针对特定行业/公司类型的扩展章节 | 条件适配（保留/删除/替换） | 技术路线分析、平台生态分析 |

### 4.2 框架要素清单

**定义**：针对特定标杆报告解构出的可复用要素集合

**包含内容**：
- 目录结构树（含层级关系）
- 特殊框架清单（含适用条件）
- 总起句功能类型库
- 论证逻辑模式库
- 语言风格参数集

### 4.3 版本管理规则

```
主版本号.次版本号.修订号（如v1.2.3）

- 主版本号+1：固定框架重大调整
- 次版本号+1：特殊框架库扩展
- 修订号+1：风格参数微调
```

## 六、最佳实践

### 6.1 标杆报告选择

- 选择**行业头部公司**的报告作为标杆
- 优先选择**券商研报**（分析框架成熟）
- 确保报告**结构完整**（8大固定框架齐全）

### 6.2 框架库维护

- 定期回顾迭代记录（建议每10次）
- 保持版本更新日志完整
- 及时将验证有效的特殊框架转正

### 6.3 质量保障

- 阶段三一致性评分≥85分视为通过
- 出现红色项（单项<70分）必须修订
- 保持参数测量的一致性和可追溯性
