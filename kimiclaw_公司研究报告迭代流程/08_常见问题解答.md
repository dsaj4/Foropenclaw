# 常见问题解答（FAQ）

---

## 一、流程理解类

### Q1：这个流程和传统的"模板写作"有什么区别？

**A**：

| 维度 | 传统模板写作 | 本迭代流程 |
|------|-------------|-----------|
| **来源** | 人工预设固定模板 | 从优质报告自动提取 |
| **灵活性** |  rigid，难以适应不同公司 | flexible，条件适配特殊框架 |
| **进化性** | 静态，不会自我改进 | 动态，每次迭代积累新经验 |
| **风格复刻** | 仅复刻结构 | 复刻结构+风格+论证方式 |
| **可持续性** | 越用越陈旧 | 越用越完善 |

**核心区别**：传统模板是"死"的，本流程是"活"的——它会学习、会进化、会积累。

---

### Q2：为什么需要四个阶段？不能简化吗？

**A**：四阶段设计遵循"解构-复现-验证-沉淀"的认知闭环：

```
阶段一（解构）：理解"好报告为什么好"
阶段二（复现）：用理解的标准写新报告
阶段三（验证）：确认新报告达到标准
阶段四（沉淀）：把验证有效的经验保存下来
```

**简化的代价**：
- 跳过阶段一 → 不理解标准，写作盲目
- 跳过阶段三 → 质量无保障，错误会重复
- 跳过阶段四 → 无法积累，每次都从零开始

---

### Q3："框架"具体指什么？包含哪些内容？

**A**：框架包含三个层次：

**结构层**：
- 章节体系（一级/二级/三级标题）
- 固定框架（必含章节）
- 特殊框架（条件适配章节）

**内容层**：
- 总起句功能类型（宣告/概括/过渡）
- 论证逻辑模式（归纳/演绎/比较）
- 信息密度配比（事实:观点比例）

**风格层**：
- 句长分布（平均句长、波动范围）
- 术语密度（专业术语占比）
- 客观度评分（主观表达程度）

## 二、执行操作类

### Q4：如何选择第一份标杆报告？

**A**：选择标准：

1. **行业代表性**：选择行业头部公司
2. **结构完整性**：包含8大固定框架
3. **分析规范性**：券商研报优于自媒体
4. **时效性**：近1-2年的报告

**推荐清单**：

| 行业 | 推荐标杆 | 理由 |
|------|---------|------|
| 互联网 | 腾讯、阿里 | 业务多元，框架完整 |
| 消费 | 茅台、五粮液 | 财务分析典范 |
| 新能源 | 宁德时代、比亚迪 | 成长型分析模板 |
| 医药 | 恒瑞医药、药明康德 | 研发驱动型框架 |
| 金融 | 招商银行、平安 | 价值型分析模板 |

---

### Q5：特殊框架的适用条件怎么判断？

**A**：每个特殊框架都有明确的适用条件：

| 特殊框架 | 核心适用条件 | 判断方法 |
|---------|-------------|---------|
| 技术路线分析 | 研发投入占比>10% | 查看研发费用/营业收入 |
| 平台生态分析 | 双边/多边市场 | 是否有供需双方互动 |
| 品牌渠道分析 | C端消费品牌 | 是否面向终端消费者 |
| 国际化战略 | 海外收入>20% | 查看地区收入拆分 |
| 周期分析 | 强周期行业 | 行业是否有明显周期 |
| 转型战略 | 业务转型期 | 是否有明确转型动作 |

**判断原则**：宁可少选，不要错选。不确定时先不选，后续可补充。

---

### Q6：总起句功能类型怎么选择？

**A**：根据段落内容性质选择：

| 内容性质 | 推荐功能类型 | 示例场景 |
|---------|-------------|---------|
| 结论先行 | 宣告型 | 投资要点、核心结论 |
| 多维拆解 | 概括型 | 财务分析、业务拆解 |
| 承上启下 | 过渡型 | 章节衔接处 |

**选择技巧**：
- 不确定时用**概括型**（最通用）
- 想强调结论时用**宣告型**
- 上下文紧密相关时用**过渡型**

---

### Q7：信息密度比例怎么控制？

**A**：按章节类型设定基准：

| 章节类型 | 事实:观点 | 实操方法 |
|---------|----------|---------|
| 财务分析 | 7:3 | 7句数据+3句解读 |
| 行业趋势 | 3:7 | 3句现象+7句推演 |
| 竞争格局 | 5:5 | 一句数据一句解读 |

**快速检查法**：
- 读完一段，问自己："这段话主要是数据还是观点？"
- 数据为主 → 检查是否有足够观点支撑
- 观点为主 → 检查是否有数据支撑

---

### Q8：句长怎么快速调整？

**A**：

**太长（>24字）→ 拆分**：
```
原句：公司2023年实现营业收入100亿元同比增长20%净利润15亿元（28字）
拆分：公司2023年实现营业收入100亿元，同比增长20%；净利润达15亿元。
```

**太短（<20字）→ 合并或扩充**：
```
原句：公司营收增长。主要得益于产品销量提升。（两句共18字）
合并：公司营收增长主要得益于产品销量的显著提升。

扩充：公司营收增长。→ 公司营收实现稳健增长，增速超出市场预期。
```

## 三、质量保障类

### Q9：一致性评分低于70分怎么办？

**A**：

**Step 1：识别红色项**
- 查看哪个维度得分<70分

**Step 2：针对性修订**

| 红色项 | 修订重点 |
|--------|---------|
| 结构保真 | 检查固定框架是否遗漏 |
| 逻辑保真 | 检查段落间衔接是否顺畅 |
| 密度保真 | 调整事实:观点比例 |
| 风格保真 | 校准句长/术语/客观度 |
| 功能适配 | 调整总起句功能类型 |

**Step 3：重新校验**
- 仅校验红色项，通过即可

---

### Q10：客观度怎么快速提升？

**A**：

**禁用词替换表**：

| 禁用（强主观） | 替换为（客观） |
|---------------|---------------|
| 必然 | 有望 |
| 肯定 | 预计 |
| 毫无疑问 | 数据显示 |
| 显然 | 值得注意的是 |
| 绝对 | 分析指出 |

**快速检查法**：
- 搜索"我们认为"、"显然"、"无疑"
- 全部替换为"数据显示"、"值得注意的是"

---

### Q11：信源标注有什么快速方法？

**A**：

**财务数据**：统一标注 `[来源：公司20XX年报]`

**行业数据**：
- 有明确来源 → 标注具体来源
- 无明确来源 → 标注 `[来源：第三方统计]`

**预测数据**：必须标注假设 `[基于XX假设]`

**批量处理技巧**：
- 写作时边写边标注
- 完成后统一搜索数字，检查是否都有来源

## 四、框架库管理类

### Q12：什么时候应该更新框架库版本？

**A**：

| 触发条件 | 版本变化 | 示例 |
|---------|---------|------|
| 新增特殊框架 | 次版本号+1 | v1.0 → v1.1 |
| 特殊框架验证转正 | 次版本号+1 | v1.1 → v1.2 |
| 参数批量优化 | 修订号+1 | v1.1.0 → v1.1.1 |
| 固定框架调整 | 主版本号+1 | v1.x → v2.0 |

**建议频率**：
- 小更新：每2-3次报告分析
- 大更新：每季度或积累10次迭代

---

### Q13：新发现的特殊框架一定要加入框架库吗？

**A**：不一定，需要评估：

**加入条件**（满足3项以上）：
- [ ] 在多份报告中出现
- [ ] 适用条件清晰可判定
- [ ] 与现有框架不重复
- [ ] 有公开数据支撑
- [ ] 综合评分≥3.0

**处理建议**：
- 5项全选 → 立即加入
- 3-4项选中 → 加入候选库，标记待验证
- ≤2项选中 → 暂不加入，作为参考

---

### Q14：框架库版本太多怎么管理？

**A**：

**文件组织**：
```
/框架库
  ├── 报告结构框架_v1.0.md    # 稳定版本
  ├── 报告结构框架_v1.1.md    # 当前版本
  ├── CHANGELOG.md            # 变更日志
  │
  └── /历史版本
      ├── 报告结构框架_v1.0.md
      └── ...
```

**使用原则**：
- 日常写作使用**当前版本**
- 重要项目锁定**稳定版本**
- 实验性项目可使用**候选版本**

## 五、进阶应用类

### Q15：可以混合多个标杆报告的框架吗？

**A**：可以，但需要谨慎：

**推荐做法**：
1. 先完整分析每个标杆报告
2. 对比各报告的框架差异
3. 选择最适合目标公司的框架组合
4. 在阶段四统一沉淀

**注意事项**：
- 不要简单拼凑，要确保逻辑一致性
- 记录框架来源，便于追溯
- 混合后的框架需要额外验证

---

### Q16：怎么处理跨行业的公司？

**A**：跨行业公司（如阿里=电商+云+金融）需要组合框架：

**处理方法**：
1. 识别公司涉及的主要行业
2. 从特殊框架库选择各行业适用框架
3. 组合成新的第五章（特殊分析）

**示例**：
```
第五章：特殊分析
├── 5.1 平台生态分析（电商框架）
├── 5.2 技术路线分析（云业务框架）
└── 5.3 合规与风控（金融框架）
```

---

### Q17：怎么评估框架库的健康度？

**A**：监控以下指标：

| 指标 | 健康阈值 | 异常处理 |
|------|---------|---------|
| 平均一致性评分 | ≥80分 | 低于阈值检查框架库 |
| 框架复用率 | ≥70% | 低于阈值检查适用条件 |
| 修订返工率 | ≤20% | 高于阈值检查阶段二质量 |
| 新框架采纳率 | 30-50% | 过高可能框架库不完善 |

**评估周期**：每10次迭代批量分析

## 六、故障排除类

### Q18：阶段二写作时感觉"套不上"框架怎么办？

**A**：可能原因及解决方案：

| 可能原因 | 解决方案 |
|---------|---------|
| 标杆报告选择不当 | 换一份业务更相似的公司 |
| 特殊框架筛选错误 | 重新评估适用条件 |
| 目标公司太特殊 | 记录为新特殊框架候选 |
| 数据不足 | 简化对应章节，标注数据缺口 |

---

### Q19：一致性校验总是某个维度不达标怎么办？

**A**：

| 不达标维度 | 可能原因 | 解决方案 |
|-----------|---------|---------|
| 结构保真 | 遗漏固定框架 | 对照清单检查 |
| 逻辑保真 | 总起句选择不当 | 调整功能类型 |
| 密度保真 | 写作习惯问题 | 刻意练习比例控制 |
| 风格保真 | 语言习惯差异 | 使用句长检查工具 |
| 功能适配 | 功能类型选择错误 | 重新匹配内容性质 |

---

### Q20：沉淀阶段发现没什么可沉淀的怎么办？

**A**：正常现象，说明：

1. **框架库已成熟**：现有框架覆盖了大部分场景
2. **本次分析常规**：没有遇到特别的新情况
3. **标杆报告普通**：没有特别突出的新框架

**处理建议**：
- 记录本次迭代的常规性
- 继续积累，等待突破性发现
- 关注参数变化趋势，批量优化
